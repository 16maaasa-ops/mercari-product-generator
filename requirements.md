# 要件定義書

## 1. プロジェクト概要
- 背景:　自分のメルカリのお店では、出品するたびに商品名と説明文を手で書いている。あいさつ文やハッシュタグなど毎回同じ部分も打ち直していて、商品名の40文字制限にも毎回引っかかる。今はGeminiのGemに同じ作業をやらせているが、これをちゃんと自分のWebアプリとして作り直したい。
- 目的:　商品の写真と実寸を渡すだけで、メルカリにそのまま貼れる「商品名」と「説明文」が出てくる状態を作る。
出てきた文章をその場で少し直して、コピーして貼るだけで出品が終わるようにして、出品にかかる時間を減らす。
- スコープ:
含むもの:　写真＋実寸から「商品名」「説明文」を作る機能。出力の編集とコピー。
含まないもの:　価格の提案（スタート価格・値下げ戦略）、複数商品のまとめ処理、商品の保存機能。

## 2. ユーザーストーリー
As a 個人利用ユーザー（自分）,
I want 商品の写真と実寸を渡すだけで商品名と説明文が作られること,
So that 毎回イチから文章を書く手間をなくせる.


## 3. 機能要件
| ID | 機能名 | 説明 | 優先度 | Phase |
|---|---|---|---|---|
| F-01 | 画像アップロード | 商品写真を複数枚アップできる（最大5枚。全体・タグ・ダメージなど） | 高 | MVP |
| F-02 | 実寸の入力 | 自由に書ける欄に「着丈65cm 身幅52cm」のように入力する | 高 | MVP |
| F-03 | 商品名の生成 | 検索されやすいワードを入れた商品名を作る（40文字以内） | 高 | MVP |
| F-04 | 文字数カウンター | 商品名の文字数を表示（例:38/40）し、40字を超えたら赤く警告する | 高 | MVP |
| F-05 | 説明文の生成 | 決められた固定フォーマットに沿った説明文を作る（下の補足参照） | 高 | MVP |
| F-06 | 出力の編集 | 生成された商品名・説明文を画面上で直接直せる | 高 | MVP |
| F-07 | コピーボタン | 商品名・説明文それぞれに別々のコピーボタンをつける | 高 | MVP |
| F-08 | エラー表示・再試行 | 生成に失敗したらエラーを表示し、「もう一度」ボタンで再実行できる | 中 | MVP |

補足：説明文の固定フォーマット（F-05）

・あいさつ（毎回この固定文で始める）

ご覧いただきありがとうございます！
即購入大歓迎です◎

値引き交渉、フォロー割も実施しております。ご質問等ありましたらコメント欄へお願いいたします(^^)
#古着屋36 ←他にも多数出品しておりますので、ご覧ください‼️


・本文のルール

魅力ポイントは「✅」を先頭につけた箇条書きにする（3〜5個）
サイズの数字のうしろには必ず「cm」をつける
サイズの直後に「※素人採寸のため、多少の誤差はご了承ください。」を入れる


・説明文の最後（固定）

大項目: ジャンル（アメカジ / Y2K / スポーツウェア / フォーマル など）
中項目: 商品の種類（フーディー / ニット / ジャケット など）
小項目: デザイン・柄・状態などの詳細
検索用キーワードを半角スペース区切りで並べる（「#」は使わない）


・守ってほしいこと

丁寧で、買う人が安心できる誠実なトーンで書く
写真から分からない実寸は勝手に数字を作らず、[着丈〇〇cm] のような空欄で返す


## 4. 非機能要件
- パフォーマンス: ページ自体は3秒以内で開く。文章の生成は画像を読む分だけ時間がかかる（数秒〜十数秒）ので、その間は「生成中」のぐるぐる表示を出す。
- セキュリティ: AIを呼ぶための鍵（APIキー）は画面側に出さず、サーバー側（環境変数）で管理する。
公開URLは誰でも開けてしまうため、他人に勝手に使われてAI利用料がかさむのを防ぐ簡単な仕組み（合言葉での入場制限など）を検討する。
- 可用性:　自分ひとりの利用が前提なので、特別な稼働率の保証は設けない。Vercelの無料枠が動いていれば十分。
- スケーラビリティ:自分ひとりの利用が前提なので、特別な稼働率の保証は設けない。Vercelの無料枠が動いていれば十分。

## 5. 制約条件
- 予算:　基本は無料枠で完結（Vercelの無料枠）。ただしClaude APIは使うたびに少額かかるので、金額は作りながら確認する。
- 期間: 2 週間
- 技術: Next.js + Vercel + Claude API
